# [CS/General] LLVM 컴파일러 인프라의 이해

> **LLVM(Low Level Virtual Machine)은 컴파일러를 모듈화하여 만들기 위한 강력한 프레임워크이다.**


## 1. LLVM이 혁명적인 이유 (N * M 문제 해결)
- 과거에는 언어가 N개이고 CPU 종류가 M개라면, 총 N * M개의 독립적인 컴파일러를 만들어야 하는 엄청난 비효율이 발생했다.
  (ex: C언어-Intel용, C언어-ARM용 따로 개발)
- LLVM은 이 공정을 3단계로 분리하여, 새로운 언어나 CPU가 추가될 때마다 **N + M번**의 개발만 하도록 아키텍쳐를 혁신했다

## 2. LLVM의 핵심 3단계 파이프라인
1. 프론트엔드
- 작성된 소스 코드(C, C++, SWfit)의 구문/의미 분석을 수행한다.
- 분석이 끝난 코드를 LLVM의 공통 언어인 LLVM IR(Intermediate Representation)로 변환한다.
2. 미들엔드(옵티마이저)
- 프론트엔드가 만든 LLVM IR을 가져와서 타겟 하드웨어와 상관없이 최적화를 수행한다.
- LLVM IR의 SSA(정적 단일 할당) 특성을 활용해, 불필요한 연산을 지우고, 반복문을 효율적으로 풀어서 코드를 가장 빠르고 가볍게 압축한다.
3. 백엔드
- 최적화된 IR을 가져와서,실제 CPU의 아키텍처에 맞춰 명령어 선택 및 레지스터 할당을 거쳐 최종 기계어(어셈블리어)로 번역한다.

## 3. 전공 과목과의 연결
- 프로그래밍 언어론(PLT):
  새로운 언어를 기획할 때 번역기를 처음부터 다 만들 필요가 없다.
  내 언어를 LLVM IR로 바꾸는 프론트엔드 규칙만 설계하면, LLVM이 제공하는 최적화와 기계어 번역 기능을 거저 사용할 수 있다.
- 컴퓨터구조:
  새로운 프로세서(ex: RISC-V)가 발명되어도 LLVM IR을 RISC-V 기계어로 번역해주는 '백엔드' 모듈 하나만 추가하면
  LLVM 인프라를 쓰는 모든 언어들을 새 CPU에서 즉시 구동할 수 있다.
