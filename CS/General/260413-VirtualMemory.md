# 🧠 메모리 구조 핵심 정리 (Virtual Memory, TLB, Locality)

## 1. 가상 주소 (Virtual Address)

### 📌 정의
- 프로그램이 사용하는 주소
- 실제 물리 메모리(RAM)의 주소가 아님

---

## ❓ 왜 가상 주소를 사용할까?

### 1️⃣ 프로그램 간 메모리 격리
- 각 프로그램은 **자기만의 메모리 공간**을 가짐
- 같은 주소를 써도 실제로는 다른 물리 메모리에 매핑됨

    프로그램 A: 0x1000 → 물리 0xA000  
    프로그램 B: 0x1000 → 물리 0xF000  

👉 서로 충돌하지 않음

---

### 2️⃣ 메모리 보호 (보안 + 안정성)
- OS가 접근 권한을 제어함
  - 읽기만 가능
  - 쓰기 가능
  - 접근 금지

👉 잘못된 접근 시 프로그램 강제 종료 (Crash)

---

### 3️⃣ 메모리 확장 효과
- RAM이 부족하면 일부 데이터를 디스크로 내림
- 필요할 때 다시 RAM으로 올림

👉 프로그램 입장에서는  
**RAM보다 더 큰 메모리를 사용하는 것처럼 보임**

---

## 2. 주소 변환 과정

CPU는 직접 물리 주소를 쓰지 않음

    가상 주소 → 물리 주소 → 메모리 접근

👉 이 변환 과정이 필요함

---

## 3. TLB (Translation Lookaside Buffer)

### 📌 정의
- 가상 주소 → 물리 주소 변환 결과를 저장하는 **캐시**

---

### 📌 동작

- TLB Hit
  - 이미 변환된 주소 있음
  - 빠르게 접근 가능

- TLB Miss
  - 변환 정보 없음
  - 페이지 테이블 조회 필요 → 느림

---

### 📌 핵심

👉 TLB = “주소 변환 캐시”

---

## 4. 데이터 지역성 (Locality)

### 📌 정의
CPU는 **가까운 데이터를 더 빠르게 접근**한다

---

### 1️⃣ 공간 지역성 (Spatial Locality)
- 근처 데이터가 함께 사용됨

    for i in array {
        print(i)
    }

👉 배열은 메모리에 연속 저장 → 빠름

---

### 2️⃣ 시간 지역성 (Temporal Locality)
- 최근 사용한 데이터를 다시 사용

    count += 1
    count += 1

👉 캐시에 남아있어 빠름

---

## 5. TLB 히트율 (TLB Hit Rate)

### 📌 정의
- TLB에서 주소를 바로 찾은 비율

---

### 📌 중요성

- 히트율 높음 → 빠름
- 히트율 낮음 → 느림

---

### 📌 데이터 지역성과 관계

- 데이터가 연속됨 → 같은 페이지 사용
- → TLB 재사용 가능
- → 히트율 증가

---

## 6. 페이지 (Page)

- 메모리를 나누는 단위 (보통 4KB)

---

## 7. 페이지 폴트 (Page Fault)

### 📌 정의
- 필요한 데이터가 RAM에 없는 상태

---

### 📌 발생 과정

1. CPU가 데이터 요청
2. RAM에 없음
3. 디스크에서 가져옴
4. RAM에 적재 후 실행

---

### 📌 문제점

👉 디스크 접근 = 매우 느림 (수천 ~ 수만 배 차이)

---

## 8. 전체 흐름 정리

    가상 주소 사용
    → 물리 주소로 변환 필요
    → TLB에서 빠르게 찾기 시도
    → 없으면 페이지 테이블 조회
    → 그래도 없으면 (RAM에 없음)
    → 페이지 폴트 발생 → 디스크 접근

---

## 🎯 핵심 요약

- 가상 주소 = 메모리 격리 + 보호 + 확장
- TLB = 주소 변환 캐시
- 데이터 지역성 = 성능 핵심
- 페이지 폴트 = 최악의 성능 저하 요인

---
