# [TIL] AI 에이전트 프레임워크: Codex, cmux, 그리고 omx

## 🚀 개요
단순한 프롬프트 엔지니어링을 넘어, AI 에이전트의 워크플로우를 시스템적으로 제어하고 상태를 유지하는 프레임워크와 **하네스 엔지니어링(Harness Engineering)**의 개념을 정리합니다.

---

## 🛠 주요 구성 요소

### 1. Codex (The Brain)
- **역할**: CLI 기반의 코드 특화 LLM 실행 엔진.
- **특징**: 터미널 명령어로 직접 모델을 호출하며, 파일 구조 분석 및 리팩토링 수행에 최적화되어 있습니다.

### 2. cmux (The Manager)
- **역할**: 에이전트 오케스트레이션 도구.
- **특징**: 터미널 멀티플렉싱 기술을 활용하여 다수의 독립된 AI 에이전트 세션을 병렬로 구성하고 각 세션에 역할을 배분합니다.

### 3. omx (The Framework)
- **풀네임**: Oh My Codex
- **역할**: 에이전트 상태 영속화 및 팀 단위 자동화 워크플로우 강제 프레임워크.
- **핵심 기능**:
    *   **State Management**: `.omx` 디렉토리에 작업 로그와 상태를 저장하여 장기 작업 시 문맥(Context) 유지 보장.
    *   **Agentic Workflow**: `$team` 명령으로 기획, 개발, 테스트 에이전트를 동시 구동하는 협업 구조 생성.

---

## 🏗 하네스 엔지니어링 (Harness Engineering)

### 개념 정의
프롬프트 엔지니어링이 모델의 **'응답 품질'**에 집중한다면, **하네스 엔지니어링**은 모델의 **'행동 반경'**을 규정하고 시스템적 가드레일을 구축하여 **예측 가능성**을 확보하는 데 집중합니다.

| 구분 | 프롬프트 엔지니어링 | 하네스 엔지니어링 |
| :--- | :--- | :--- |
| **주요 목적** | 답변의 정확도 및 품질 향상 | 모델 출력을 제어하는 구조적 제약 설계 |
| **핵심 수단** | 지시문(Instruction) 최적화 | 시스템 가드레일 및 자동 강제 시스템 |
| **지향점** | 나은 답변 얻기 | 실수가 불가능한(Error-proof) 환경 구축 |

### 주요 메커니즘
1.  **영속적 컨텍스트 관리**: 컨텍스트 부패(Context Decay) 방지를 위해 매 세션마다 참조되는 영속적 컨텍스트 파일(`.md`)을 운영하여 정보 손실과 규칙 위반을 원천 차단합니다.
2.  **자동 피드백 루프**: 린터(Linter), 프리커밋 훅(Pre-commit Hook)과 같은 강제 시스템을 통해 AI가 스스로 실수를 교정하도록 유도합니다.
3.  **가비지 컬렉션(GC)**: 주기적으로 불량 코드 패턴을 관리하여 시스템 전반의 품질을 유지합니다.

---

## 💡 결론
AI 에이전트 시스템에서 가장 중요한 것은 자유도가 아니라 **통제된 예측 가능성**입니다. 하네스 엔지니어링은 AI가 시스템의 규칙 내에서만 움직이도록 '굴레(Harness)'를 씌워 기술적 부채를 최소화하고 개발 환경의 신뢰도를 극대화하는 핵심 기술입니다.
