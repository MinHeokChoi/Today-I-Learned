## 1. 점근 표기법

### 핵심 정의

점근 표기법은 알고리즘의 실행 시간을 입력 크기 n의 함수로 표현하는 방법이다. 교수님 시험에서는 **직접 증명**과 **극한 증명** 두 가지 방식 모두 출제된다.

| 표기법 | 의미 | 수식 조건 |
|---|---|---|
| `f(n) ∈ O(g(n))` | g(n)이 f(n)의 **상한** | `f(n) ≤ c·g(n)` for all n ≥ N |
| `f(n) ∈ Ω(g(n))` | g(n)이 f(n)의 **하한** | `f(n) ≥ c·g(n)` for all n ≥ N |
| `f(n) ∈ Θ(g(n))` | g(n)이 f(n)의 **상한이자 하한** | O이고 Ω이면 Θ |
| `f(n) ∈ o(g(n))` | g(n)이 f(n)을 **압도적으로** 초월 | `lim f(n)/g(n) = 0` |

### 극한과 표기법의 관계

```
lim(n→∞) g(n)/f(n) = c > 0  →  g(n) ∈ Θ(f(n))
lim(n→∞) g(n)/f(n) = 0      →  g(n) ∈ o(f(n))
lim(n→∞) g(n)/f(n) = ∞      →  f(n) ∈ o(g(n))
```

### 증명 형식 — 정의 기반 직접 증명

> **예제: f(n) = n² + 3n³ ∈ Θ(n³) 증명** (21년 기출)

**[O(n³) 증명]**

n ≥ 0인 모든 정수 n에 대해서, `n² ≤ n³`이 성립하므로,

```
n² + 3n³ ≤ n³ + 3n³ = 4n³
```

따라서 c = 4, N = 0을 선택하면 `n² + 3n³ ≤ 4n³`이 성립한다.  
그러므로 `f(n) ∈ O(n³)`이다.

**[Ω(n³) 증명]**

n ≥ 0인 모든 정수 n에 대해서, `n² ≥ 0`이 성립하므로,

```
n² + 3n³ ≥ 3n³
```

따라서 c = 3, N = 0을 선택하면 `n² + 3n³ ≥ 3n³`이 성립한다.  
그러므로 `f(n) ∈ Ω(n³)`이다.

**[결론]**

`f(n) ∈ O(n³)`이고 `f(n) ∈ Ω(n³)`이므로, `f(n) ∈ Θ(n³)`이다. □

### 증명 형식 — 극한 기반 증명 (small-o)

> **예제: n²/2 ∈ o(n³) 증명** (23년 기출)

```
lim(n→∞) (n²/2) / n³ = lim(n→∞) 1/(2n) = 0
```

극한값이 0이므로 `n²/2 ∈ o(n³)`이다. □

### ⚠️ 시험 주의사항

같은 차수끼리는 small-o 관계가 절대 성립하지 않는다. `f(n) ∈ Θ(g(n))`이면 `f(n) ∉ o(g(n))`이다. 예를 들어 `lim(n→∞) n³/n³ = 1 ≠ 0`이므로 `n³ ∉ o(n³)`이다.

## 2. 재현식과 복잡도 도출

### 재현식이란?

재현식(Recurrence)은 알고리즘의 실행 시간을 **자기 자신으로 표현한 식**이다. 알고리즘 구조에서 세 가지만 확인하면 바로 세울 수 있다.

```
1. 재귀 호출이 몇 번? → 항의 개수
2. 각 호출의 입력 크기가 얼마? → 각 항의 인자
3. 재귀 호출 외 나머지 작업 비용? → 마지막 항
```

### 주요 알고리즘별 재현식

| 알고리즘 | 재현식 | 복잡도 |
|---|---|---|
| 피보나치 (순수 재귀) | `T(n) = T(n-1) + T(n-2)` | `Ω(2^(n/2))` |
| 합병 정렬 | `W(n) = 2W(n/2) + n-1` | `Θ(n log n)` |
| 퀵 정렬 (평균) | `T(n) = 2T(n/2) + n` | `O(n log n)` |
| 퀵 정렬 (최악) | `T(n) = T(0) + T(n-1) + n-1` | `O(n²)` |
| Strassen 행렬 | `T(n) = 7T(n/2) + O(n²)` | `O(n^log₂7)` |

### 마스터 방법 (Master Method)

분할정복 형태의 재현식 `T(n) ≤ aT(n/b) + O(n^d)`를 풀어주는 공식이다.

```
a = 재귀 호출 횟수
b = 문제 크기가 줄어드는 비율
d = 재귀 호출 외 나머지 작업의 차수

Case 1: a = b^d  →  T(n) = O(n^d · log n)
Case 2: a < b^d  →  T(n) = O(n^d)
Case 3: a > b^d  →  T(n) = O(n^(log_b a))
```

> **예제: 합병 정렬에 마스터 방법 적용**
>
> `W(n) = 2W(n/2) + n-1`에서 a=2, b=2, d=1이므로 `b^d = 2^1 = 2 = a`  
> → Case 1에 해당 → `W(n) = O(n log n)`

### ⚠️ 마스터 방법이 적용 안 되는 경우

피보나치처럼 `T(n) = T(n-1) + T(n-2)`는 하위 문제가 **n/b가 아니라 n-1, n-2처럼 상수만큼 줄어드는 형태**이므로 마스터 방법을 적용할 수 없다.

## 3. 동적 계획법

### DP의 핵심 아이디어

DP는 **상향식(bottom-up) 접근법**을 사용한다. 가장 작은 하위 문제의 답을 먼저 구하여 저장해두고, 더 큰 문제를 풀 때 저장된 값을 꺼내 쓴다. 따라서 중복 계산이 발생하지 않는다.

DP를 적용할 수 있는 조건은 **최적 부분 구조(Optimal Substructure)**다. 전체 문제가 최적이면 그 하위 문제도 최적이어야 한다는 성질이다.

### 분할정복 vs 동적 계획법 비교

| | 분할정복 | 동적 계획법 |
|---|---|---|
| 접근 방식 | 하향식 (Top-down) | 상향식 (Bottom-up) |
| 중복 계산 | 허용 (재귀 호출) | 제거 (저장 후 재사용) |
| 대표 알고리즘 | 합병 정렬, 퀵 정렬 | 연쇄 행렬 곱셈, 최적 BST |

---

### 연쇄 행렬 곱셈

#### 문제 정의

n개의 행렬을 곱할 때, **곱셈 횟수가 최소가 되는 묶음 순서**를 찾는 문제다. 행렬 A(p×q)와 B(q×r)를 곱하는 데 필요한 곱셈 횟수는 `p × q × r`번이다.

#### 점화식

`M[i][j]` = Aᵢ부터 Aⱼ까지 곱할 때의 **최소 곱셈 횟수**

```
M[i][i] = 0  (base case: 행렬 1개는 곱셈 불필요)

M[i][j] = min_{i ≤ k < j} ( M[i][k] + M[k+1][j] + d_{i-1} × d_k × d_j )
```

여기서 `d_{i-1} × d_k × d_j`는 왼쪽 덩어리와 오른쪽 덩어리를 **최종으로 합치는 비용**이다. Aᵢ의 크기가 `d_{i-1} × dᵢ`이기 때문에 Aᵢ의 행 수가 `d_{i-1}`이다.

#### 테이블 채우는 순서

구간 길이 1 → 2 → 3 → ... 순서로 채운다. 큰 구간을 계산할 때 작은 구간의 값이 필요하기 때문이다.

#### 예제 계산 (25년 기출)

> A₁(5×2), A₂(2×3), A₃(3×4), A₄(4×6)  
> d₀=5, d₁=2, d₂=3, d₃=4, d₄=6

**구간 길이 2:**

```
M[1][2]: k=1 → M[1][1] + M[2][2] + d₀×d₁×d₂ = 0 + 0 + 5×2×3 = 30
M[2][3]: k=2 → M[2][2] + M[3][3] + d₁×d₂×d₃ = 0 + 0 + 2×3×4 = 24
M[3][4]: k=3 → M[3][3] + M[4][4] + d₂×d₃×d₄ = 0 + 0 + 3×4×6 = 72
```

**구간 길이 3:**

```
M[1][3]:
  k=1: M[1][1] + M[2][3] + d₀×d₁×d₃ = 0 + 24 + 5×2×4 = 64
  k=2: M[1][2] + M[3][3] + d₀×d₂×d₃ = 30 + 0 + 5×3×4 = 90
  → min(64, 90) = 64  (P[1][3] = 1)

M[2][4]:
  k=2: M[2][2] + M[3][4] + d₁×d₂×d₄ = 0 + 72 + 2×3×6 = 108
  k=3: M[2][3] + M[4][4] + d₁×d₃×d₄ = 24 + 0 + 2×4×6 = 72
  → min(108, 72) = 72  (P[2][4] = 3)
```

**구간 길이 4:**

```
M[1][4]:
  k=1: M[1][1] + M[2][4] + d₀×d₁×d₄ = 0 + 72 + 5×2×6 = 132
  k=2: M[1][2] + M[3][4] + d₀×d₂×d₄ = 30 + 72 + 5×3×6 = 192  ← 212아님 주의
  k=3: M[1][3] + M[4][4] + d₀×d₃×d₄ = 64 + 0 + 5×4×6 = 184
  → min(132, 192, 184) = 132  (P[1][4] = 1)
```

**완성된 M 테이블:**

| M[i][j] | 1 | 2 | 3 | 4 |
|---|---|---|---|---|
| 1 | 0 | 30 | 64 | **132** |
| 2 | | 0 | 24 | 72 |
| 3 | | | 0 | 72 |
| 4 | | | | 0 |

**최적 순서 역추적 (P 테이블):**

```
P[1][4] = 1 → A₁ | (A₂A₃A₄)
P[2][4] = 3 → A₁ | ((A₂A₃)A₄)
P[2][3] = 2 → A₁((A₂A₃)A₄) ✓
```

최소 곱셈 횟수: **132**, 최적 순서: **A₁((A₂A₃)A₄)**

### 최적 이분 검색 트리

#### 문제 정의

n개의 키와 각 키의 검색 확률이 주어졌을 때, **평균 검색 비교 횟수가 최소**인 BST를 구하는 문제다. BST에서 키를 찾는 비교 횟수는 해당 노드의 **깊이 + 1**이다.

#### 핵심 직관

검색 확률이 높은 키일수록 루트에 가깝게 배치해야 평균 비교 횟수가 줄어든다.

#### 점화식

`A[i][j]` = keyᵢ부터 keyⱼ까지로 구성된 최적 BST의 **평균 검색 비교 횟수**

```
A[i][i] = pᵢ  (base case: 키 1개, 루트 자신 비교 1번 × 확률)
A[i][i-1] = 0  (빈 트리)

A[i][j] = min_{i ≤ k ≤ j} ( A[i][k-1] + A[k+1][j] + Σpₘ (m=i to j) )
```

마지막 항 `Σpₘ`이 붙는 이유는, keyₖ를 루트로 선택하는 순간 서브트리 안의 모든 키들이 **한 층씩 내려가기** 때문이다. 깊이가 1 늘어나면 비교 횟수도 1 늘어나므로 그 비용이 `Σpₘ`이다.

#### 예제 계산

> p₁=0.5, p₂=0.3, p₃=0.2

**구간 길이 1 (base case):**

```
A[1][1] = 0.5,  A[2][2] = 0.3,  A[3][3] = 0.2
```

**구간 길이 2:**

```
A[1][2]: Σp = 0.5+0.3 = 0.8
  k=1: A[1][0] + A[2][2] + 0.8 = 0 + 0.3 + 0.8 = 1.1
  k=2: A[1][1] + A[3][2] + 0.8 = 0.5 + 0 + 0.8 = 1.3
  → min(1.1, 1.3) = 1.1  (P[1][2] = 1, key1이 루트)

A[2][3]: Σp = 0.3+0.2 = 0.5
  k=2: A[2][1] + A[3][3] + 0.5 = 0 + 0.2 + 0.5 = 0.7
  k=3: A[2][2] + A[4][3] + 0.5 = 0.3 + 0 + 0.5 = 0.8
  → min(0.7, 0.8) = 0.7  (P[2][3] = 2, key2가 루트)
```

**구간 길이 3:**

```
A[1][3]: Σp = 0.5+0.3+0.2 = 1.0
  k=1: A[1][0] + A[2][3] + 1.0 = 0 + 0.7 + 1.0 = 1.7
  k=2: A[1][1] + A[3][3] + 1.0 = 0.5 + 0.2 + 1.0 = 1.7
  k=3: A[1][2] + A[4][3] + 1.0 = 1.1 + 0 + 1.0 = 2.1
  → min(1.7, 1.7, 2.1) = 1.7  (P[1][3] = 1 또는 2)
```

**완성된 A 테이블:**

| A[i][j] | 1 | 2 | 3 |
|---|---|---|---|
| 1 | 0.5 | 1.1 | **1.7** |
| 2 | | 0.3 | 0.7 |
| 3 | | | 0.2 |

**최적 트리 구조 (P[1][3] = 1일 때):**

```
      key1
         \
         key2
            \
            key3
```

**검증:** 1×0.5 + 2×0.3 + 3×0.2 = 0.5 + 0.6 + 0.6 = **1.7** ✓

## 4. 정렬 알고리즘

### 퀵 정렬

#### 핵심 아이디어

피벗(pivot)을 선택해 배열을 두 그룹으로 분할한다. 분할 완료 후 피벗은 **최종 정렬된 위치에 확정**된다. 이후 왼쪽/오른쪽 그룹을 독립적으로 재귀 정렬한다.

#### pl/pr 포인터 분할 규칙

```
pl: 피벗보다 큰 원소를 만날 때까지 → 오른쪽으로 이동 후 멈춤
pr: 피벗보다 작거나 같은 원소를 만날 때까지 → 왼쪽으로 이동 후 멈춤

pl < pr  → pl과 pr이 가리키는 원소 교환 후 각각 한 칸 이동, 반복
pl ≥ pr  → 반복 종료, 피벗과 pr이 가리키는 원소 교환 → 피벗 위치 확정
```

#### 복잡도

```
평균: O(n log n)  →  재현식: T(n) = 2T(n/2) + n
최악: O(n²)       →  재현식: T(n) = T(0) + T(n-1) + n-1
```

최악의 경우는 **이미 정렬된 배열**에서 첫 번째 원소를 피벗으로 선택할 때 발생한다. 매번 가장 작은(또는 큰) 원소가 피벗이 되어 분할 후 한쪽 그룹의 크기가 0, 다른 쪽이 n-1이 된다.

#### 예제 — 1차 분할 과정 (25년 기출)

> 배열: [15, 22, 13, 27, 12, 10, 20, 25], pivot = 15

```
초기:    [15, 22, 13, 27, 12, 10, 20, 25]
          ↑   ↑                       ↑
        pivot  pl                     pr

1단계: pl=22(>15), pr=10(<15) → 교환
         [15, 10, 13, 27, 12, 22, 20, 25]

2단계: pl=27(>15), pr=12(<15) → 교환
         [15, 10, 13, 12, 27, 22, 20, 25]

3단계: pl ≥ pr → 피벗과 pr(=12) 교환
         [12, 10, 13, 15, 27, 22, 20, 25]
                      ↑
                 15 위치 확정
```

이후 [12, 10, 13]과 [27, 22, 20, 25]를 각각 재귀 정렬한다.

### 합병 정렬

#### 핵심 아이디어

배열을 **정확히 절반으로** 분할한 뒤, 각 절반을 재귀적으로 정렬하고, 두 정렬된 배열을 **merge**로 합친다. 분할은 단순하게, **합치기가 핵심**이다.

#### merge 과정

두 정렬된 배열의 맨 앞 원소를 비교해서 작은 것을 결과 배열에 먼저 넣는다. 한쪽이 소진되면 나머지를 그대로 이어 붙인다. merge 비용은 **결과 배열 크기 = 비교 횟수의 상한**이므로 O(n)이다.

#### 복잡도

```
재현식: W(n) = 2W(n/2) + n-1
복잡도: Θ(n log n)
```

복잡도가 `n log n`인 이유는 **매 레벨마다 merge 비용의 합이 항상 n**이고, **레벨 수가 log n**이기 때문이다.

#### 퀵 정렬 vs 합병 정렬 비교

| | 퀵 정렬 | 합병 정렬 |
|---|---|---|
| 평균 복잡도 | O(n log n) | O(n log n) |
| 최악 복잡도 | **O(n²)** | O(n log n) |
| 추가 메모리 | O(1) (제자리 교환) | **O(n)** (임시 배열) |
| 최악 발생 조건 | 이미 정렬된 배열 | 없음 |

## 5. 유형 간 연결 관계 요약

이 수업의 핵심 개념들은 하나의 흐름으로 연결되어 있다.

**분할정복 ↔ 동적 계획법:** 분할정복은 하위 문제가 독립적이어서 재귀 호출로 해결하고, DP는 하위 문제가 중복되어 저장 후 재사용한다. 퀵 정렬/합병 정렬의 재현식(유형 B)과 연쇄 행렬 곱셈/최적 BST의 테이블 계산(유형 D)은 이 두 패러다임의 실제 표현이다.

**재현식 → 복잡도:** 알고리즘 구조를 재현식으로 옮기면(유형 B), 마스터 방법이나 직접 계산으로 복잡도를 도출할 수 있다. 이 복잡도가 곧 점근 표기법(유형 A)의 증명 대상이 된다.

**DP 패턴의 일관성:** 피보나치의 "fib(0)~fib(n)을 순서대로 한 번씩 계산", 연쇄 행렬 곱셈의 "구간 길이 1→2→3→n 순서로 테이블 채우기", 최적 BST의 "하위 트리부터 상위 트리 순서로 계산" — 이 세 가지는 모두 **bottom-up DP의 같은 패턴**이다.
