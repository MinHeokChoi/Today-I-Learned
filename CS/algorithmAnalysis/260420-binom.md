# 📚 [Algorithm] 수학적 귀납법 증명: 이항계수 재귀 알고리즘 분석

> **주제:** 수학적 귀납법(Mathematical Induction)을 이용한 알고리즘 복잡도 증명
> **목표:** 이항계수 재귀 호출 횟수가 $2\binom{n}{k} - 1$임을 논리적으로 증명하기

---

## 1. 수학적 귀납법의 3단계 구조
교수님 답안지 작성 시 반드시 지켜야 하는 표준 형식:
1.  **귀납 출발점 (Basis):** 가장 작은 단위(Base Case)에서 명제가 참임을 증명.
2.  **귀납 가정 (Inductive Hypothesis):** $n=m$일 때 명제가 성립한다고 가정.
3.  **귀납 단계 (Inductive Step):** 가정을 바탕으로 $n=m+1$일 때도 명제가 성립함을 유도.

---

## 2. 핵심 개념: 이항계수 재귀 알고리즘

### 재현식 (Recurrence Relation)
이항계수 $\binom{n}{k}$를 구하는 재귀 알고리즘의 **항의 개수(함수 호출 횟수)** $T(n, k)$는 다음과 같이 정의된다.

* **재귀식:** $T(n, k) = T(n-1, k-1) + T(n-1, k) + 1$ (단, $0 < k < n$)
    * *Note:* `+1`은 자기 자신을 호출(실행)하는 횟수 1회를 의미함.
* **기초 단계 (Base Case):** $T(n, 0) = 1, T(n, n) = 1$
    * *Note:* $k=0$이거나 $k=n$이면 재귀 없이 즉시 값을 반환하므로 호출 횟수는 1회임.

### 활용 공식: 파스칼의 법칙 (Pascal's Identity)
$$\binom{n}{k} = \binom{n-1}{k-1} + \binom{n-1}{k}$$

---

## 3. 명제 증명 과정

**[명제]** 모든 $n > 0, 0 \le k \le n$에 대하여 $T(n, k) = 2\binom{n}{k} - 1$ 이다.

### Step 1: 귀납 출발점 (Basis)
$n=1$일 때, 가능한 $k$는 0과 1이다. (알고리즘의 최소 단위)
* $k=0$일 때: $T(1, 0) = 1$. 명제 식 대입: $2\binom{1}{0} - 1 = 2(1) - 1 = 1$. (성립)
* $k=1$일 때: $T(1, 1) = 1$. 명제 식 대입: $2\binom{1}{1} - 1 = 2(1) - 1 = 1$. (성립)

### Step 2: 귀납 가정 (Inductive Hypothesis)
$n=m$일 때, $0 \le k \le m$인 모든 $k$에 대하여 다음이 성립한다고 가정한다.
$$T(m, k) = 2\binom{m}{k} - 1$$

### Step 3: 귀납 단계 (Inductive Step)
$n=m+1$일 때, 재귀가 발생하는 범위인 $0 < k < m+1$에서 명제가 성립함을 보인다.

1.  재현식 정의에 의해: $T(m+1, k) = T(m, k-1) + T(m, k) + 1$
2.  귀납 가정을 대입: 
    $$= [2\binom{m}{k-1} - 1] + [2\binom{m}{k} - 1] + 1$$
3.  수식 정리:
    $$= 2[\binom{m}{k-1} + \binom{m}{k}] - 1 - 1 + 1$$
    $$= 2[\binom{m}{k-1} + \binom{m}{k}] - 1$$
4.  파스칼의 법칙 적용:
    $$= 2\binom{m+1}{k} - 1$$

**결론:** $n=m+1$일 때도 명제가 성립하므로, 수학적 귀납법에 의해 모든 $n, k$에 대해 명제는 참이다.

---

## 💡 내가 헷갈렸던 점 & 깨달은 점 (Self-Correction)

### 1. $T(n, k)$ 재현식의 의미
* **헷갈린 점:** 처음에는 $T(n, k)$가 이항계수 '값'인 줄 알고 $+1$을 빼먹음.
* **깨달은 점:** 증명하려는 대상이 **'호출 횟수'**라면, 자기 자신을 실행하는 `+1`이 재현식에 반드시 포함되어야 함.

### 2. Base Case는 왜 항상 $n=1$이 아닐까?
* **헷갈린 점:** 보통 $n=1$만 생각했는데, 왜 $k=0, k=n$을 보는지 의문이었음.
* **깨달은 점:** Base Case는 **'재귀가 멈추는 지점'**이다. 이항계수 코드 상에서 `if (k==0 || k==n)`일 때 멈추므로, 이 지점들이 논리적 시작점이 됨.

### 3. 귀납 가정과 단계에서 왜 범위(등호)가 다를까?
* **헷갈린 점:** $n=m$일 때는 $0 \le k \le m$인데, $n=m+1$일 때는 왜 $0 < k < m+1$인지 궁금했음.
* **깨달은 점:** * **가정**은 이미 검증된 '상태'이므로 양 끝단(Base Case)을 포함한다.
    * **단계**는 '재귀 식'을 써서 증명하는 과정이므로, 재귀가 실제로 발생하는 '사이 값' 범위에서만 논리가 전개되기 때문에 양 끝단(등호)을 제외한다.

### 4. 시간 복잡도의 오해
* **헷갈린 점:** $O(n!)$으로 생각함.
* **깨달은 점:** 하나의 항을 구하기 위해 두 개의 하위 항을 재귀적으로 호출하는 구조가 반복되어, 연산량이 매 단계마다 2배씩 늘어나는 이진 트리(Binary Tree)를 형성하기 때문에 **지수 시간 $O(2^n)$**임.
