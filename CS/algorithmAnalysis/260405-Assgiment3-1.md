# [Algorithm] 동적 계획법(DP)과 그래프 경로 탐색 핵심 정리

## 1. 파스칼의 법칙 (Pascal's Identity)과 문제 해결 로직
조합(Combination)의 기본 성질인 파스칼의 법칙은 단순한 수학 공식을 넘어 컴퓨터 과학의 핵심 문제 해결 기법을 담고 있다.

* **점화식:** $\binom{n}{k} = \binom{n-1}{k-1} + \binom{n-1}{k}$ (단, $0 < k < n$)
* **기저 조건 (Base Case):** $k=0$ 이거나 $k=n$ 일 때 값은 1이 된다. 프로그래밍에서 무한 루프나 스택 오버플로우를 막는 필수적인 예외 처리와 같다.
* **재귀적 사고 (Divide & Conquer):** 특정 원소를 '포함하는 경우'와 '포함하지 않는 경우' 두 가지 하위 문제로 쪼개어 거대한 요구사항을 해결 가능한 단위로 분할한다.
* **최적화 (Dynamic Programming):** 팩토리얼 연산 시 발생하는 정수 오버플로우를 방지하기 위해 덧셈 공식을 활용하여 배열에 값을 저장(Memoization/Tabulation)하는 방식의 최적화를 수행할 수 있다.

## 2. 플로이드-워셜 알고리즘 2 (Floyd-Warshall)
모든 정점 쌍 간의 최단 경로를 구하는 알고리즘으로, 거리뿐만 아니라 '어떤 경로를 거쳐왔는지'에 대한 상태 역추적 로직을 포함한다. 라우팅 프로토콜 설계나 데이터 통신 인프라 최적화에 직접적으로 응용되는 핵심 원리다.

* **핵심 점화식 (Relaxation):** $D[i][j] = \min(D[i][j], D[i][k] + D[k][j])$
* **경로 갱신 (행렬 $D$):** 기존에 알고 있던 직행 경로와 새롭게 발견한 $k$ 우회 경로 중 더 짧은 거리로 지속해서 업데이트한다.
* **경로 역추적 (행렬 $P$):** $P[i][j] = k$는 $i$에서 $j$로 갈 때 가장 높은 인덱스의 중간 정점이 $k$라는 의미다. 
* **경로 복원 원리:** $P$ 행렬의 값을 기준으로 목적지까지의 경로를 앞 구간과 뒷 구간으로 계속 반씩 쪼개어 들어가는 재귀적 방식을 통해 전체 최단 경로를 복원해 낼 수 있다.
* **실무적 의의:** 앱 개발 시 결제 오류 화면 네비게이션 스택 추적이나 분산 서버의 로그 트레이싱(Distributed Tracing)과 같은 데이터 역추적 시스템 설계의 뼈대가 된다.

## 3. 연쇄 행렬 곱셈 (Matrix Chain Multiplication)
동적 계획법(DP)의 대표적인 예시로, 다수의 행렬을 곱할 때 연산 횟수(비용)를 최소화하는 최적의 결합 순서를 찾는다. 부분 문제의 답을 조립해 전체의 최적해를 도출하는 과정(Bottom-up)을 잘 보여준다.

* **차원 배열 $d$ 로의 압축:** 행렬 곱셈의 '꼬리 물기' 특성을 활용해 중복되는 차원 크기를 1차원 배열로 압축하여 변수 관리를 효율화한다. (예: $A_i$의 크기는 $d_{i-1} \times d_i$)
* **비용 계산 점화식:** $M[i, j] = \min_{i \le k < j} \{ M[i, k] + M[k+1, j] + d_{i-1} \cdot d_k \cdot d_j \}$
* **공식의 의미:** 임의의 지점 $k$를 기준으로 사슬을 두 덩어리로 잘랐을 때, 앞 덩어리 최적 비용($M[i, k]$) + 뒤 덩어리 최적 비용($M[k+1, j]$) + 두 결과 행렬을 곱하는 스칼라 연산 비용($d_{i-1} \cdot d_k \cdot d_j$)의 합 중 가장 작은 값을 찾는다.
* **Bottom-up DP 테이블 채우기:** 문제의 길이($L$)를 1(단일 행렬)부터 시작해 2, 3, 4... 로 점진적으로 늘려가며 부분 문제의 해를 구한다. 행렬 $M$의 대각선을 따라 값이 채워지며, 이전에 구해둔 작은 단위의 답을 재활용하여 최종 전체 곱셈 묶음의 최소 비용을 효율적으로 도출해 낸다.
