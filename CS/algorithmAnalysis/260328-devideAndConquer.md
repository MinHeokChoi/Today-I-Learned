# 분할정복 알고리즘 정리 (이분검색, 합병정렬, 빠른정렬)

## 분할정복이란?

- **분할(Divide)**: 문제를 작은 부분으로 나눈다
- **정복(Conquer)**: 각각을 해결한다
- **통합(Combine)**: 해답을 모은다 (필요한 경우)

---

## 1. 이분검색 (Binary Search)

### 동작 원리

정렬된 배열에서 mid = (low + high) / 2 위치의 값과 비교하여 탐색 범위를 절반씩 줄인다.

- `S[mid] == x` → 찾음
- `S[mid] > x` → 왼쪽 탐색 (high = mid - 1)
- `S[mid] < x` → 오른쪽 탐색 (low = mid + 1)
- `low > high` → 못 찾음

### 헷갈리기 쉬운 포인트

- mid 계산은 **정수 나눗셈** (나머지 버림)
- 종료 조건은 `low > high` — 이걸 빼먹으면 무한루프

---

## 2. 합병정렬 (Mergesort)

### 동작 원리

1. 배열을 반씩 쪼개서 원소 1개까지 분할
2. 이미 정렬된 두 부분배열을 비교하면서 합병

**핵심**: 나눌 때는 아무것도 안 하고, **합칠 때 정렬이 일어난다**



### merge 동작 

두 정렬된 배열에서 각각 i, j 포인터를 두고:
- `U[i] < V[j]`이면 U[i]를 넣고 i++
- 아니면 V[j]를 넣고 j++
- 한쪽이 소진되면 **나머지를 그대로 복사**


## 3. 빠른정렬 (Quicksort)

### 동작 원리

1. pivot(기준값) 선택 — 보통 S[low]
2. partition: pivot보다 작은 건 왼쪽, 큰 건 오른쪽으로 분할
3. pivot 위치 확정 후, 왼쪽/오른쪽에 대해 재귀

**핵심**: 합병정렬과 반대로, **나눌 때 정렬이 일어나고** 합치는 건 필요 없다

### partition 동작 (제일 중요)

```
pivotitem = S[low]
j = low                    // "작은 애들 경계"
for i = low+1 to high:
    if S[i] < pivotitem:
        j++                // 경계 넓히고
        swap(S[i], S[j])   // 작은 애를 왼쪽으로
swap(S[low], S[j])          // pivot을 제자리에
```

- **j의 역할**: pivot보다 작은 원소들의 마지막 위치를 추적
- **j를 먼저 증가시키고** 교환하는 순서 주의 (j++; swap;)
- 순회가 끝나면 S[low]와 S[j]를 교환하여 pivot 확정


### 합병정렬 vs 퀵정렬 비교 

| | 합병정렬 | 퀵정렬 |
|---|---------|---------|
| 최악 시간복잡도 | Θ(n lg n) | Θ(n²) |
| 평균 시간복잡도 | Θ(n lg n) | Θ(n lg n) |
| 제자리 정렬? | ❌ (추가 공간 Θ(n)) | ⭕ |
| 안정 정렬? | ⭕ | ❌ |
| 정렬이 일어나는 시점 | 합칠 때 (merge) | 나눌 때 (partition) |
| 실제 속도 | 비교 횟수 적음 | 데이터 이동 적어서 실제로 더 빠름 |
