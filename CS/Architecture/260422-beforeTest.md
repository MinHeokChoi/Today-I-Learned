# [TIL] 컴퓨터 구조 중간고사 핵심 요약 (RISC-V)

## 1. 2의 보수와 비트 패턴 해석
* 최상위 비트(MSB)에만 음수 가중치($-2^{n-1}$)를 주고 나머지를 양수로 더하면 비트 반전 없이 빠르게 음수 값을 계산할 수 있음.
* 이진수에서 1이 연속해서 꽉 채워진 구간의 합은 **(가장 높은 자리의 다음 자리 값) - (가장 낮은 자리 값)**으로 빠르게 도출 가능함.

## 2. RISC-V 호출 규약 (Calling Convention) 및 스택 메모리
* **Callee-saved (호출된 함수가 보존):** `s` 레지스터(`s0`~`s11`), `sp` [1-3]. 함수 내부 사용 전 기존 값을 스택에 백업 필수.
* **Caller-saved (호출하는 함수가 보존):** `t` 레지스터, `a` 레지스터, `ra` [1-3]. 다른 함수 호출(`jal`) 이후에도 값이 필요하다면 부르기 전 스스로 백업.
* 스택은 높은 주소에서 낮은 주소 방향으로 자라며 (`sp = sp - size`), 사용 후에는 다시 할당했던 공간을 반납(`sp = sp + size`)하여 `sp`를 원래대로 되돌려야 함 [4, 5].

## 3. 단일 사이클 데이터패스 (Single-Cycle Datapath)
* **임계 경로 (Critical Path):** 가장 긴 지연 시간을 가지는 경로로, 전체 프로세서의 최소 클럭 주기를 결정 [6-8]. 명령어 메모리, 레지스터, ALU, 데이터 메모리를 모두 거치는 `lw` 명령어의 실행 경로가 보통 임계 경로가 됨 [6].
* **제어 신호 분석:** 예를 들어 `lw` 명령어는 `RegWrite=1`, `MemtoReg=1`, `ALUSrc=1`, `MemWrite=0`의 제어 신호를 가짐 [9].
* **다단계 디코딩:** 메인 디코더(Main Decoder)가 `opcode`를 분석해 ALU 디코더에 명령어의 대략적 유형(ALUOp)을 전달하면, ALU 디코더가 최종 ALU 제어 신호(예: 덧셈 `010`)를 생성하여 제어 유닛을 효율적으로 구성함 [6, 10].

## 4. C 코드의 어셈블리 변환 (배열 및 반복문)
* 배열의 인덱스를 실제 메모리 주소로 변환할 때는 요소의 바이트 크기에 맞춰 논리 왼쪽 시프트(`slli`) 연산을 수행하여 오프셋을 구함 (예: 4바이트 Word 배열은 `slli reg, reg, 2`, 8바이트 배열은 `slli reg, reg, 3`) [11, 12].
* 베이스 주소(Base Address)를 가진 레지스터에 계산된 오프셋을 더해 최종 주소를 구한 뒤 메모리에 접근해야 함.
