# [TIL] 컴퓨터 구조: Sign-extension과 시프트 연산의 이해

## 1. Sign-extension (부호 확장)

### Signed와 Unsigned의 개념 차이
32비트(4바이트)의 공간이 있을 때 데이터를 해석하는 두 가지 관점이 있다.
* **Unsigned (부호 없음):** 32개 비트 전체를 오로지 양수(0 포함)의 크기를 표현하는 데 사용한다.
* **Signed (부호 있음):** 제일 왼쪽에 있는 최상위 비트(MSB) 하나를 부호(+, -)를 나타내는 데 사용한다. 나머지 31개 비트로 값의 크기를 표현한다.

### 언제 Signed로, 언제 Unsigned로 해석해야 할까?
데이터(0과 1의 나열) 자체에는 부호가 없다. **어떤 명령어(Instruction)로 읽어 들이느냐**에 따라 해석 방식이 달라진다. 

### 확장의 두 가지 방식
4비트짜리 데이터를 8비트 크기의 레지스터에 넣어야 할 때처럼, 데이터의 비트 수를 늘릴 때 빈칸을 채우는 방식이 다르다.
* **Sign Extension (부호 확장):** `Signed` 데이터를 확장할 때 사용. 자신의 최상위 비트(부호 비트)를 그대로 복사해서 빈 앞자리를 채운다. 이렇게 하면 원래의 수학적 값과 부호가 완벽히 보존된다.
* **Zero Extension (제로 확장):** `Unsigned` 데이터를 확장할 때 사용. 부호가 없으므로 빈 앞자리를 무조건 `0`으로 채워 크기를 유지한다.

## 2. 연산에서 비트를 바라보는 관점: 시프트(Shift) 연산

데이터의 비트를 밀어내는 시프트 연산은 비트를 대하는 관점에 따라 두 가지로 나뉜다.
* **산술적(Arithmetic):** 데이터를 '수학적 수치'로 대한다는 뜻.
* **논리적(Logical):** 데이터를 '독립적인 스위치(비트)들의 나열'로 대한다는 뜻.

> **💡** 왼쪽 시프트(Left Shift)는 둘 다 오른쪽 빈자리를 0으로 채우므로 동일하게 동작한다. **두 연산의 진짜 차이는 '오른쪽 시프트(Right Shift)'를 할 때 발생한다.**

### 1) 산술적 시프트 (Arithmetic Shift): "수학적인 의미를 지켜라"
부호가 있는 정수를 2로 나눌 때(오른쪽 시프트), 단순히 비트만 밀어버리면 최상위 비트가 0이 되어 음수가 양수로 바뀌는 대참사가 일어난다. 
* **특징:** 값이 가진 수학적 의미(크기와 부호)를 끝까지 보존하며 올바른 나눗셈 연산을 수행하기 위해, 밀어내고 생긴 빈자리를 **원래의 부호 비트(MSB)로 채워 넣는다.**

### 2) 논리적 시프트 (Logical Shift): "기계적으로 스위치만 옮겨라"
데이터를 수학적인 숫자로 보지 않고, 32개의 스위치가 나열된 것으로만 본다. 
* **특징:** 기계적으로 스위치를 옆으로 밀어내고, 새롭게 생긴 빈 공간은 가장 기본 상태인 **`0`으로 채워 넣는다.** 데이터의 내용보다는 '논리적인 비트의 이동'이라는 기계적 조작에만 집중한다.

### 3) 각각 Signed, Unsigned의 짝꿍이 된 이유
* **부호 있는 정수(Signed) ➔ 산술적 시프트:** 음수라는 수학적 개념을 지켜야 올바른 나눗셈이 성립하므로, 부호를 유지해 주는 산술적 시프트를 사용한다.
* **부호 없는 정수(Unsigned) ➔ 논리적 시프트:** 애초에 부호 비트라는 개념이 없고 모든 비트가 순수하게 크기만 나타낸다. 따라서 기계적으로 0을 채워 넣어도 부호가 망가질 걱정이 없으며, 양수의 나눗셈이 깔끔하게 이루어진다.
