# [TIL] 프로그래밍 언어론: EBNF부터 연산 의미론까지

## 1. EBNF (Extended BNF)
기존 BNF의 복잡한 재귀 연산과 긴 규칙을 간결하고 가독성 높게 표현하기 위해 도입된 확장 문법 요소.

* **`[ ]` (선택적 요소):** 생략 가능함 (0번 또는 1번 등장)을 의미.
* **`( )` 와 `|` (그룹화 및 대안):** 소괄호 안에서 여러 요소 중 하나를 선택.
* **`{ }` (반복):** 0번 이상 무한히 반복될 수 있음을 의미하며, BNF의 재귀(Recursion)를 효과적으로 대체.
* **최근 변형:** 화살표 대신 콜론(`:`)을 사용하거나, 선택적 요소에 `opt`, 다중 선택에 `oneof` 키워드를 직접 사용하기도 함.

## 2. 정적 의미론 (Static Semantics)
CFG(문맥 자유 문법)만으로는 표현하기 어렵거나 불가능한 **컴파일 타임의 문맥적 제약 조건**을 다루는 영역. 실제 프로그램의 실행 의미와는 무관함.

* **CFG의 한계 (처리하기 힘든 구조들):**
  * 변수가 사용되기 전 선언되어야 한다는 규칙 (Non-context-free).
  * 수식 내 피연산자 간의 타입 일치 여부 검사 (Context-free하지만 모든 경우의 수를 적으려면 표현이 지나치게 복잡해짐).

## 3. 속성 문법 (Attribute Grammars, AGs)
정적 의미론(타입 검사 등)을 명시하고 컴파일러가 이를 확인할 수 있도록 CFG에 속성, 평가 함수, 술어(Predicate)를 추가한 형식적 정의.

* **속성의 3가지 종류:**
  1. **합성 속성 (Synthesized Attributes):** 자식 노드들의 값을 바탕으로 계산되어 부모 노드로 전달되는 상향식(Bottom-up) 정보.
  2. **상속 속성 (Inherited Attributes):** 부모나 형제 노드로부터 전달받아 자식 노드로 내려가는 하향식/횡적 정보 (예: `expected_type`).
  3. **내장 속성 (Intrinsic Attributes):** 트리의 단말 노드(잎사귀)에 초기부터 주어져 있는 기본적인 속성값 (예: 심볼 테이블에서 가져온 변수의 실제 타입).
* **평가 과정:** 트리를 구성한 후, 상향식과 하향식이 결합된 방식으로 트리를 오르내리며 속성값을 계산하고 술
