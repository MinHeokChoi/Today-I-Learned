# Chapter 3. 구문(Syntax)과 의미론(Semantics)

> **핵심 질문:** "우리가 만든 프로그래밍 언어를 컴퓨터(컴파일러)가 어떻게 이해하게 만들 것인가?"

이 질문에 답하기 위해 언어를 **형태(Syntax)** 와 **의미(Semantics)** 두 축으로 나누어 분석한다.

---

## 1부. 구문 (Syntax) — "코드가 어떻게 생겼는가?"

컴퓨터가 코드를 글자 그대로 읽고, **문법적으로 올바른 문장인지** 구조를 파악하는 단계다.

### BNF & EBNF

- 언어의 문법을 정의하는 **"조립 설명서"**
- BNF(Backus-Naur Form)는 생성 규칙(production rule)의 집합으로 문법을 기술한다.
- EBNF는 BNF에 반복(`{}`)·선택(`[]`)·그룹(`()`) 표기를 추가하여 더 간결하게 표현한다.

### 파스 트리 (Parse Tree)

- 조립 설명서(BNF)대로 코드를 분해해 본 **계층적 구조도**
- 루트부터 리프까지 문법 규칙이 어떻게 적용되었는지를 시각적으로 보여 준다.

### 모호성 (Ambiguity)

- 같은 코드에 대해 **파스 트리가 2개 이상** 그려지는 치명적인 문제
- 해결 방법: **연산자 우선순위(precedence)** 와 **결합 법칙(associativity)** 을 문법 규칙에 반영한다.

---

## 2부. 정적 의미론 (Static Semantics) — "실행 전, 논리적으로 말이 되는가?"

문법 구조가 올바르더라도 "정수형 변수에 문자열을 대입" 같은 논리적 오류가 없는지 **컴파일 시점**에 검사한다.

### CFG의 한계

- BNF 같은 문맥 자유 문법(CFG)만으로는 변수의 타입 일치 여부나 선언 여부를 확인할 수 없다.

### 속성 문법 (Attribute Grammars)

- 파스 트리의 각 노드에 **타입(Type) 같은 정보(속성)** 를 부착한다.
- **합성 속성(Synthesized):** 자식 → 부모로 정보가 올라간다.
- **상속 속성(Inherited):** 부모 → 자식으로 정보가 내려간다.
- 이 속성들이 트리를 타고 흐르며 **타입 불일치** 등의 에러를 컴파일 단계에서 걸러낸다.

---

## 3부. 동적 의미론 (Dynamic Semantics) — "실제로 실행하면 무슨 일이 일어나는가?"

에러 없이 완벽한 코드가 **실제 기계에서 동작할 때** 어떤 의미를 갖는지, 세 가지 관점으로 정의한다.

### 1. 연산 의미론 (Operational Semantics)

- 코드를 직접 실행하여 **기계 상태(메모리·레지스터)의 변화**를 관찰한다.
- 직관적이지만 특정 기계에 종속적이라는 한계가 있다.

### 2. 표시 의미론 (Denotational Semantics)

- 기계를 배제하고, **순수한 수학적 함수**로 프로그램의 입력 상태 → 출력 상태를 매핑한다.
- 가장 추상적이고 형식적이며, 그만큼 복잡하다.

### 3. 공리 의미론 (Axiomatic Semantics)

- 논리학(Hoare Logic)을 이용해 코드가 **올바르게 동작함을 수학적으로 증명**한다.
- **전제 조건(Precondition):** 실행 전에 반드시 참이어야 하는 조건
- **사후 조건(Postcondition):** 실행 후에 보장되는 조건
- **루프 불변식(Loop Invariant):** 반복문이 도는 동안 항상 참인 조건

---

## 한 줄 요약

> 코드를 **문법 규칙(BNF)** 으로 조립하고 → 실행 전에 **타입 검사(속성 문법)** 를 마친 뒤 → 실행될 때의 동작을 **관찰(연산) / 함수화(표시) / 증명(공리)** 하는 여정이다.
