# [TIL] 프로그래밍 언어의 문법 체계와 파싱 (BNF, CFG, LL vs LR)

## 1. BNF: 프로그램을 생성해내는 '메타 언어'
- **개념:** 프로그래밍 언어의 구문을 기술하기 위한 **언어를 설명하는 언어(Meta-language)**입니다.
- **핵심 철학:** 문법은 단순한 규칙의 나열이 아니라, 시작 기호(Start Symbol)에서 출발하여 **완전한 프로그램 단위를 생성해내는 체계(Generator)**입니다.
- **작동 원리:** 비단말 기호(추상적 단위)를 단말 기호(실제 코드)로 치환해 나가는 **'유도(Derivation)'** 과정을 통해 소스코드가 탄생합니다.

---

## 2. Context-Free Grammar (CFG): '주변 눈치' 안 보는 자유
- **왜 '자유(Free)'인가?** 어떤 기호를 변환할 때 주변 환경(문맥)을 살필 필요 없이 독립적으로 규칙을 적용할 수 있기 때문입니다.
- **장점:**
    - **단순함:** 규칙의 왼쪽(LHS)에 기호가 딱 하나만 오므로 구조가 명확합니다.
    - **효율성:** 컴파일러가 앞뒤 상황을 다 따지지 않아도 되어 파싱 속도가 빠릅니다.
    - **확장성:** 새로운 기능을 '조립 블록'처럼 추가하기 쉽습니다.

---

## 3. 모호성 해결: 계급이 정해주는 유일한 해석
- **문제점:** 동일한 코드에 대해 파싱 트리가 2개 이상 만들어지면 연산 순서가 꼬입니다. (예: `3 + 4 * 5`)
- **해결책:** BNF 규칙을 여러 단계(`E`, `T`, `F`)로 쪼개어 **연산의 우선순위(Precedence)**를 부여합니다.
- **결과:** 재귀의 깊이를 조절함으로써, 소스코드를 보고 **단 하나의 유일한 파싱 트리**를 생성하도록 강제합니다.

---

## 4. 파서(Parser)의 세계: LL vs LR
컴파일러가 문법을 '인식'하는 방식은 크게 두 가지로 나뉩니다.

### 🔍 LL 파서 (Top-down) : "공부용 & 직관적"
- **방식:** 시작 기호부터 아래로 쪼개 내려가며(**Top-down**) 소스코드와 맞추어 봅니다.
- **특징:**
    - **장점:** 사람이 이해하기 매우 쉽고, 코드로 직접 구현(Recursive-Descent)하기 좋습니다.
    - **단점:** 좌재귀(Left Recursion) 규칙이 있으면 무한 루프에 빠집니다. 처리할 수 있는 문법 범위가 좁습니다.
- **Why?:** "규칙이 실제 코드로 어떻게 변하는가"를 눈으로 확인하며 기초 체력을 기르기에 최적입니다.

### 🚀 LR 파서 (Bottom-up) : "실전용 & 강력함"
- **방식:** 소스코드의 단어들을 먼저 보고, 이를 규칙에 맞춰 거꾸로 합쳐가며(**Reduce**) 올라갑니다.
- **특징:**
    - **장점:** 거의 모든 CFG를 처리할 수 있을 만큼 강력합니다. 좌재귀도 문제없이 소화합니다.
    - **단점:** 사람이 손으로 파싱 테이블을 만들기 불가능할 정도로 복잡합니다. (Yacc, Bison 같은 도구 사용 필수)
- **Why?:** 복잡한 현대 언어의 문법을 제약 없이 가장 빠르고 안정적으로 처리하는 '근육질 엔진'이기 때문입니다.

| 구분 | LL 파서 | LR 파서 |
| :--- | :--- | :--- |
| **방식** | Top-down (하향식) | Bottom-up (상향식) |
| **구현** | 비교적 쉬움 (재귀적 하강) | 매우 어려움 (자동 생성 도구 활용) |
| **강점** | 직관성, 디버깅 용이 | 강력한 표현력, 성능 |
| **특이사항** | 좌재귀 해결 필수 | 좌재귀 허용 |

---

## 5. 결론: 구문 분석(Syntax) vs 의미 분석(Semantics)
1. **구문 분석:** BNF를 바탕으로 **'어순'**과 **'구조'**가 맞는지 검증하여 파싱 트리를 생성합니다. (컴파일러의 첫 번째 관문)
2. **의미 분석:** 트리가 완성된 후, "어순은 맞는데 말이 되는가?"(예: 타입 체크, 변수 선언 여부 등)를 검사합니다.

> **한 줄 요약**
> BNF는 **"문법적으로 올바른 문장을 인식하기 위한 완벽한 설계도"** 역할을 수행한다.
