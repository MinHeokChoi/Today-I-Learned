# [PL] 3장. 구문(Syntax)과 의미론(Semantics) 기술의 기초

## 1. 언어 정의의 두 축
프로그래밍 언어의 명세는 크게 두 가지로 나뉜다.
* **구문 (Syntax):** 표현식, 문장, 프로그램 단위의 형태나 구조를 의미한다. (예: `var x = 10` 이라는 작성 형식)
* **의미론 (Semantics):** 그러한 표현식이나 문장들이 갖는 실제적인 의미와 동작을 뜻한다. (예: 10이라는 값을 메모리 공간 x에 할당하라)

## 2. 어휘 분석 단위: Lexeme과 Token
컴파일러의 어휘 분석기(Lexical Analyzer)가 코드를 읽어 들이는 최소 단위이다.
* **어휘 항목 (Lexeme):** 언어의 가장 낮은 수준의 구문 단위로, 소스 코드에 적힌 실제 문자열 그 자체를 뜻한다. (예: `*`, `sum`, `begin`)
* **토큰 (Token):** 어휘 항목들이 속하는 논리적인 카테고리(분류)이다. (예: 식별자, 키워드, 연산자, 리터럴)

## 3. 구문 기술의 표준: BNF (Backus-Naur Form)
1959년 존 배커스가 발명한 표기법으로, 문맥 자유 문법(Context-Free Grammars)과 동일한 표현력을 가지는 메타 언어이다.

### 3.1. BNF의 기본 요소
* **비단말 기호 (Nonterminals):** 구문 구조의 추상적인 클래스를 나타내며 보통 `< >`로 둘러싸서 표기한다.
* **단말 기호 (Terminals):** 더 이상 분해되지 않는 최종적인 어휘 항목이나 토큰이다.
* **규칙 (Rule):** 왼쪽(LHS)에는 하나의 비단말 기호가 오고, 오른쪽(RHS)에는 단말 기호와 비단말 기호들의 혼합된 문자열이 온다. (예: `<stmt> -> <var> = <expr>`)

### 3.2. 핵심 기법
* **선택 (Alternative RHSs):** 수직 바(`|`)를 사용하여 하나의 기호가 가질 수 있는 여러 대안을 나타낸다.
* **재귀 (Recursion):** 규칙 안에 자기 자신을 포함하여 리스트와 같은 반복 구조를 설명한다. 

## 4. 유도(Derivation)와 파스 트리(Parse Tree)
* **유도 (Derivation):** 시작 기호에서 출발하여 규칙을 반복적으로 적용해 최종 문장(모두 단말 기호인 상태)을 만들어내는 과정이다. 
  * 각 단계에서 가장 왼쪽의 비단말 기호를 확장하면 '좌측 유도(Leftmost derivation)'라 부른다.
* **파스 트리 (Parse Tree):** 유도 과정을 한눈에 파악할 수 있도록 나타낸 계층적인 그래프(트리) 표현이다. 컴파일러의 구문 분석기가 코드를 올바르게 이해했는지 검증하는 구조도 역할을 한다.

## 5. 모호성(Ambiguity)과 해결책
* **모호성 정의:** 어떤 문법이 단 하나의 문장에 대해 **두 개 이상의 서로 다른 파스 트리**를 생성한다면, 그 문법은 모호한(Ambiguous) 문법이다. 이는 컴파일러가 코드를 여러 의미로 해석하게 만들어 치명적인 오류를 낳는다.
* **모호성 해결 방법 (비모호한 문법 설계):**
  1. **연산자 우선순위 (Precedence):** 파스 트리의 형태를 이용해 우선순위를 지시한다. 먼저 계산되어야 하는 연산자(예: `/`)를 나타내는 규칙을 파스 트리의 더 아래쪽(하위 계층)에 배치하여 모호성을 제거한다.
  2. **결합 법칙 (Associativity):** 문법의 재귀 방향(좌측 재귀 또는 우측 재귀)을 통해 동일한 연산자가 연속될 때의 계산 방향을 지시한다. (예: `<expr> -> <expr> + const`는 좌측 결합)
