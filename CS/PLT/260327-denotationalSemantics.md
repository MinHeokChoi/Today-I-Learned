# 표시 의미론 (Denotational Semantics)

> **핵심 철학:** 프로그램의 의미를 기계의 동작이 아닌, **순수한 수학적 함수**로 정의한다.
> "이 코드는 수학적으로 이런 함수식과 완벽히 똑같다"를 증명해내는 작업이다.

---

## 1. 표시 의미론의 본질

1970년 Scott과 Strachey가 개발한 방법으로, 재귀 함수 이론(recursive function theory)에 기반한다. 세 가지 의미론 중 **가장 추상적인** 접근법이며, 프로그램의 모든 구성 요소를 수학적 객체(mathematical objects)에 대응시킨다.

---

## 2. 핵심 도구 — 상태(State)와 VARMAP

### 상태(State)

특정 시점에 프로그램이 가진 **모든 변수와 그 값의 집합**이다.

```
s = { <i₁, v₁>, <i₂, v₂>, ..., <iₙ, vₙ> }
    (i = 변수명,  v = 현재 값)
```

예시: 변수 x에 3, y에 5가 들어있다면 → `s = { <x, 3>, <y, 5> }`

### VARMAP 함수

변수명과 상태를 넣으면 해당 변수의 현재 값을 꺼내주는 도우미 함수다.

```
VARMAP(x, s) = 3
VARMAP(y, s) = 5
```

---

## 3. 단계별 매핑 시뮬레이션

작은 단위부터 시작해 점점 복잡한 문장으로 함수를 확장해 나간다.

### 1단계: 10진수 매핑 — M_dec

소스 코드에 적힌 **문자 '9'는 아직 숫자가 아니라 텍스트**다. 이를 수학적 정수로 바꾸는 함수다.

```
M_dec('0') = 0,  M_dec('1') = 1,  ...,  M_dec('9') = 9    ← 기저 조건
M_dec(<dec_num> 'd') = 10 × M_dec(<dec_num>) + d           ← 재귀 조건
```

예시 — M_dec('127') 계산:

```
M_dec('1')   = 1                          ← 한 글자, 기저 조건
M_dec('12')  = 10 × M_dec('1') + 2 = 12  ← 재귀
M_dec('127') = 10 × M_dec('12') + 7 = 127 ← 재귀
```

문자열 `'127'`이 수학적 정수 **127**로 매핑된다.

### 2단계: 수식 매핑 — M_e

`a + 2` 같은 수식을 받아 **정수(Z) 또는 error**로 매핑한다.

#### 먼저, 수식의 BNF 문법 규칙

M_e가 다루는 수식(Expression)은 다음 BNF로 정의된다.

```
<expr> → <dec_num> | <var> | <binary_expr>
<binary_expr> → <expr> <op> <expr>
<op> → + | - | * | /
```

즉 수식은 세 가지 형태 중 하나다:
- **숫자 리터럴** (`<dec_num>`): `127`, `3` 같은 숫자 텍스트
- **변수** (`<var>`): `x`, `y` 같은 변수명
- **이항 연산식** (`<binary_expr>`): `x + 3`, `a * b` 같이 두 수식을 연산자로 연결한 것

#### M_e의 수학적 정의 (경우별)

```
① 숫자 리터럴인 경우:
   M_e(<dec_num>, s) Δ= M_dec(<dec_num>)
   → 문자 '127'을 M_dec으로 넘겨 정수 127로 변환한다.

② 변수인 경우:
   M_e(<var>, s) Δ= 
       if VARMAP(<var>, s) == undef
           then error
           else VARMAP(<var>, s)
   → 장부(s)에서 변수의 현재 값을 꺼낸다. 선언되지 않은 변수면 error.

③ 이항 연산식인 경우:
   M_e(<expr₁> <op> <expr₂>, s) Δ=
       if   M_e(<expr₁>, s) == error  OR  M_e(<expr₂>, s) == error
           then error
       else if <op> == '+'  then M_e(<expr₁>, s) + M_e(<expr₂>, s)
       else if <op> == '-'  then M_e(<expr₁>, s) - M_e(<expr₂>, s)
       else if <op> == '*'  then M_e(<expr₁>, s) × M_e(<expr₂>, s)
       else if <op> == '/'  then
           if M_e(<expr₂>, s) == 0
               then error                          ← 0으로 나누기 방지
               else M_e(<expr₁>, s) / M_e(<expr₂>, s)
   → 왼쪽·오른쪽을 각각 재귀적으로 계산한 뒤, 연산자(<op>)에 맞게 결합한다.
     어느 한쪽이라도 error면 전체가 error.
```

#### 예시 — `x + 3 * y` 계산 (s = {x:2, y:4})

수식 트리 구조 (연산자 우선순위 적용: `*`가 `+`보다 먼저):

```
M_e(x + 3*y, s)
 = M_e(x, s) + M_e(3*y, s)                      ← ③ 이항 연산 (+)
 = VARMAP(x, s) + M_e(3*y, s)                    ← ② 변수
 = 2 + M_e(3*y, s)
 = 2 + (M_e(3, s) × M_e(y, s))                  ← ③ 이항 연산 (*)
 = 2 + (M_dec('3') × VARMAP(y, s))              ← ① 숫자, ② 변수
 = 2 + (3 × 4)
 = 2 + 12
 = 14
```

핵심: M_e는 수식을 만나면 **"이게 숫자야? 변수야? 연산식이야?"** 를 판단한 뒤, 재귀적으로 쪼개어 최종 정수 값 하나로 합쳐낸다.

### 3단계: 대입문 매핑 — M_a

`x := E`가 실행되면 **기존 상태 s를 복사하여, x의 값만 바꾼 새 상태 s'을 만든다.**

```
M_a(x := E, s) Δ=
    if M_e(E, s) == error
        then error
        else s' = { <i₁,v₁'>, <i₂,v₂'>, ..., <iₙ,vₙ'> }
            where  if iⱼ == x  then vⱼ' = M_e(E, s)    ← 타겟 변수만 새 값
                   else             vⱼ' = VARMAP(iⱼ, s)  ← 나머지는 그대로 복사
```

예시 — `x := y + 1` 실행 (s = {x:3, y:5}):

```
M_e(y + 1, s) = 5 + 1 = 6
s' = { <x, 6>, <y, 5> }       ← x만 6으로 바뀌고 y는 그대로
```

### 4단계: 반복문 매핑 — M_l

while 루프의 **반복(iteration)을 재귀(recursion)로 변환**하여 정의한다.

```
M_l(while B do L, s) Δ=
    if M_b(B, s) == undef   then error           ← 조건식 계산 불가
    else if M_b(B, s) == false  then s            ← 조건 거짓 → 루프 종료, 현재 상태 반환
    else if M_sl(L, s) == error then error        ← 본체 실행 중 에러
    else M_l(while B do L, M_sl(L, s))            ← 본체 한 번 실행 후 자기 자신 재귀 호출
```

여기서 **M_sl(L, s)** 는 루프 본체(문장 리스트 L)를 현재 상태 s에서 한 번 쭉 실행한 결과 상태를 돌려주는 함수다.

예시 — `while (x > 0) do x = x - 1` (s = {x:3}):

```
M_l(while, {x:3})
 └→ M_sl 실행 → {x:2}  →  M_l(while, {x:2})       ← 1회차
      └→ M_sl 실행 → {x:1}  →  M_l(while, {x:1})   ← 2회차
           └→ M_sl 실행 → {x:0}  →  M_l(while, {x:0}) ← 3회차
                └→ x>0 거짓! → {x:0} 반환              ← 종료
```

## 4. 평가 및 비교

### vs 연산 의미론 (Operational Semantics)

| 구분 | 연산 의미론 | 표시 의미론 |
|---|---|---|
| 도구 | 코딩된 알고리즘으로 상태 변화를 **관찰** | 엄밀한 수학적 함수로 상태 변화를 **정의** |
| 기계 의존성 | 가상 기계에 종속적 | 기계와 완전히 독립적 |

### 장점

프로그램의 정확성(correctness)을 수학적으로 증명할 수 있고, 언어 설계의 엄밀한 기초를 제공한다.

### 단점

수식이 매우 복잡해서 일반 프로그래머나 컴파일러 개발자에게는 실용성이 떨어진다.

---

## 한 줄 요약

> 표시 의미론은 프로그램의 모든 것을 **텍스트 → 숫자(M_dec) → 수식 계산(M_e) → 상태 변환(M_a) → 루프의 재귀적 상태 변환(M_l)** 순서로 수학 함수에 매핑하는 여정이다.
