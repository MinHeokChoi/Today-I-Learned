### 오늘 배운 핵심

#[260311] 프로그래밍 언어론의 핵심 요소

### 1. 설계 (design)
- 정의: 인간의 논리를 컴퓨터가 수행 가능한 계산 모델로 정의하고 약속하는 단계.
- 배우는 것
  - 구문론과 의미론: 문법의 형태와 실제 동작의 의미 정의
  - 바인딩 & 스코프: 이름(Name)과 값(Value)가 연결되는 시점과 유효 범위 결정.
 
### 2. 안전 (Saftey)
- 정의: 프로그램이 정의되지 않은 동작을 하지 않도록 보장하여 시스템 붕괴를 방지
- 배우는 것
  - 타입 시스템: 컴파일 타임 및 런타임에 데이터 형식의 일치 여부 검증.
  - 메모리 안정성: 잘못된 메모리 접근을 원천 차단.

### 3. 추상화 (Abstraction)
- 정의: 하드웨어의 물리적 복잡성을 숨기고, 문제를 해결하는 논리적 단위로 모델링.
- 배우는 것
  - 데이터/제어 추상화: 함수, 클래스, 모듈 등을 통한 복잡성 관리.
  - 다형성: 동일성 인터페이스로 다양한 형태의 데이터를 처리하는 유연성 확보.

### 4. 변환 (Transformation)
- 정의: 인간 친화적인 언어를 특절 실행 환경이 이해하는 저수준 지시서로 효율적으로 번역
- 배우는 것
  - 컴파일/인터프리팅: 소스 코드를 실행 파일 또는 중간 코드로 변환.
  - 최적화 & IR: 중간 표현(Intermediate Representation, IR)을 활용해 타겟 하드웨어에 최적화된 명령어를 생성.
