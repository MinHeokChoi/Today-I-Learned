# [TIL] SwiftUI 기초: View 프로토콜, body, 그리고 레이아웃 (ZStack vs overlay)

## 1. SwiftUI 화면 구성의 3가지 핵심 규칙

### ① View 프로토콜 (화면 등장 자격증 🪪)
- SwiftUI에서 화면에 무언가를 띄우려면 반드시 `View` 프로토콜을 준수해야 한다.
- 구조체(Struct) 뒤에 `: View`를 붙이는 것은 클래스 상속이 아니라, "화면에 그려질 자격이 있다"는 선언(약속)이다.

### ② body 프로퍼티와 `some View` (작업반장과 오마카세 👨‍🍳)
- **`body`**: 화면에 무엇을 그릴지 결정하고 결과물을 뱉어내는 필수 프로퍼티.
- **`some View`**: 반환되는 뷰의 정확하고 복잡한 타입 이름(예: `VStack<TupleView<...>>`)을 숨기고, "어떤 종류인진 몰라도 확실한 View 자격증을 가진 결과물이야!"라고 시스템에 알려주는 키워드.

### ③ 단일 뷰 반환 규칙과 Stack (택배 상자 법칙 📦)
- `body`는 **반드시 단 1개의 뷰(결과물)**만 반환해야 한다.
- 여러 개의 뷰를 화면에 띄우려면 `VStack`, `HStack`, `ZStack` 같은 '상자'에 담아서 하나의 뷰로 묶어 제출해야 한다.
- *Tip:* 최신 SwiftUI에서는 `@ViewBuilder`라는 자동 포장기가 있어서 Stack 없이 나열해도 에러는 안 나지만, 원하는 대로(가로/세로 등) 예쁘게 배치하려면 반드시 Stack 명시가 필요하다.

---

## 2. 뷰를 겹치는 두 가지 방법: ZStack vs overlay

### ① ZStack (평등한 팬케이크 🥞)
- **개념:** 화면의 안쪽(바닥)부터 바깥쪽으로 뷰를 차곡차곡 쌓아 올린다. 먼저 작성한 코드가 밑바탕에 깔린다.
- **관계:** 안에 들어있는 뷰들은 모두 평등한 관계다.
- **크기 결정:** 내부 뷰들 중 **가장 크기가 큰 뷰**를 기준으로 전체 `ZStack`의 크기가 늘어난다.

### ② .overlay() (주인공과 스티커 ✨)
- **개념:** 이미 존재하는 바탕 뷰(주인공) 위에 다른 뷰(스티커)를 찰싹 붙이는 수식어(Modifier).
- **관계:** 바탕 뷰와 얹혀지는 뷰 사이에 확실한 부모-자식(주종) 관계가 있다.
- **크기 결정:** 전체 뷰의 크기는 **무조건 바탕(부모) 뷰의 크기**를 따른다. 스티커(overlay)가 아무리 커도 바탕 뷰의 영역을 억지로 늘리지 못한다.




# [TIL] SwiftUI : return문 언제?

## 1. return 키워드, 언제 생략하고 언제 써야 할까? 💡

### ✅ 생략 가능한 경우 (Implicit Return)
- `body` 안에서 **오직 하나의 뷰**만 반환할 때.
- `if-else`나 `switch` 같은 조건문만 있을 때 (자동 포장기 `@ViewBuilder`가 작동함).
- 예: `var body: some View { Text("안녕") }`

### 🚨 반드시 명시해야 하는 경우
- 뷰를 반환하기 전에 **변수 선언, 계산, 로그 출력(`print`) 등 '다른 코드'**가 섞여 있을 때.
- 시스템에게 "이게 진짜 최종 결과물이야!"라고 이정표를 세워줘야 하기 때문.
- 예: 
  ```swift
  var body: some View {
      let name = "Gemini"
      return Text(name) // return 필수!
  }


# [TIL] SwiftUI: Struct Initializer의 비밀 (Memberwise vs Manual)

## 1. 멤버와이즈 초기화 (Memberwise Initializer) 🎁
- **개념:** Swift 구조체(`struct`)는 내부 변수들만 선언해두면, 초기화 코드(`init`)를 자동으로 만들어준다.
- **특징:** 단순히 외부에서 받은 값을 그대로 변수에 담기만 할 때 사용하며, 코드를 아주 간결하게 유지해준다.

## 2. 수동 `init`을 직접 작성해야 하는 경우 🛠️
컴퓨터가 자동으로 해주는 것 이상의 **'특별한 로직'**이 필요할 때 사용한다.

### ① 데이터 전처리 (Processing)
- 데이터를 받아서 저장하기 직전에 형태를 바꾸고 싶을 때.
- 예: 숫자를 받아서 "20세"처럼 문자열로 가공하여 저장하는 경우.
  ```swift
  struct AgeView: View {
      let ageString: String

      init(age: Int) {
          self.ageString = "\(age)세" // 숫자를 문자로 가공!
      }

      var body: some View { Text(ageString) }
  }
### ② 값의 변형 및 기본값 설정
- 소문자를 대문자로 바꾸거나, 조건에 따라 다른 값을 초기값으로 설정하고 싶을 때.

### ③ 접근 권한 설정
- 다른 모듈이나 파일에서 접근할 수 있도록 public init 등을 명시해야 할 때.
